## 计划概要

| 阶段 | 时间 | 主题 |
|------|------|------|
| **Phase 0** | 第 1–2 月 | 概率统计（数学理论）→ Python 量化栈 |
| **Phase 1** | 第 3–6 月 | 时间序列、回归、最优化、机器学习入门 |
| **Phase 2** | 第 7–10 月 | 多因子、价量、回测与风控（含期权/固收/行为/资产配置等扩展） |
| **Phase 3** | 第 11–14 月 | 完整策略 pipeline + 竞赛/实盘 |

建议每周投入约 **12–15 小时**。详见 [LEARNING_PLAN.md](LEARNING_PLAN.md)。

## 知识库结构速览

```
knowledge/
├── phase0_基础与环境/      # 概率论、数理统计、python_量化栈
├── phase1_数学与统计深化/  # 线性代数、时间序列、回归、最优化、机器学习入门
├── phase2_金融与因子/      # 市场基础、多因子、价量、回测、风险与组合；期权、固收、行为、资产配置（扩展）
├── phase3_策略与实证/      # 策略类型、过拟合控制、实盘与系统、论文与研报
└── 速查与汇总/             # 各阶段要点速查、推荐书籍与资源
```

- **按阶段学习**：依 LEARNING_PLAN 顺序阅读对应 phase 下文档。  
- **速查与复习**：各阶段要点与公式见 `knowledge/速查与汇总/`，入口说明见 [knowledge/速查与汇总/README.md](knowledge/速查与汇总/README.md)。  
- **完整列表**：见 [KNOWLEDGE_FILES.md](KNOWLEDGE_FILES.md)。

## 数学公式显示

knowledge 下文档使用 **LaTeX** 公式（`$$...$$` 与 `\(...\)`）。若公式显示为**乱码或原始代码**，多半是当前 Markdown 预览未启用数学渲染。

**解决**：在 VS Code/Cursor 中安装 **Markdown+Math** 等支持 KaTeX 的扩展后再打开预览；或用 **Typora、Obsidian** 打开 `.md`。GitHub 网页不渲染 .md 中的公式，建议本地用上述方式查看。

详细说明见 **[knowledge/公式显示说明.md](knowledge/公式显示说明.md)**。
