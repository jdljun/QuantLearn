# 回测框架

> 对应学习计划 Phase 2：收益率计算、滑点与手续费、前视偏差、幸存者偏差、样本内/外。

**前言**：回测是验证策略逻辑与历史表现的工具；若收益计算、成本或时间线有误，结果会失真甚至虚高。本章按教材体例组织：**收益计算与净值 → 成本与滑点 → 前视偏差 → 幸存者偏差 → 样本内/外划分 → 检查清单**，每节遵循定义 → 性质/公式 → 例题/实现 → 注意，便于自检与报告规范。

## 第 1 章 收益率计算

**动机**：回测的核心是「按日或按周期计算组合收益并累成净值」；收益定义（简单 vs 对数）、分红复权、权重与调仓方式都会影响结果，必须统一约定，否则不同模块或不同人之间的结果不可比。

### 1.1 单期收益

**定义 1.1.1（简单收益率）**  
设 \(P_{t-1}\)、\(P_t\) 为相邻两期价格，**简单收益率**为
\[
R_t = \frac{P_t - P_{t-1}}{P_{t-1}} = \frac{P_t}{P_{t-1}} - 1.
\]

**定义 1.1.2（对数收益率）**  
**对数收益率**为
\[
r_t = \ln\frac{P_t}{P_{t-1}} = \ln P_t - \ln P_{t-1}.
\]

**性质 1.1.1（多期收益）**  
- 简单收益：多期总收益为 \((1+R_1)(1+R_2)\cdots(1+R_T) - 1\)。  
- 对数收益：多期总对数收益为 \(r_1 + r_2 + \cdots + r_T = \ln(P_T/P_0)\)；在 \(R_t\) 较小时 \(r_t \approx R_t\)。

**定义 1.1.3（组合收益）**  
若权重 \(w_i\)、标的简单收益 \(R_{i,t}\)，则**组合收益**为
\[
R_{p,t} = \sum_i w_{i,t} R_{i,t} = \boldsymbol{w}_t^\top \boldsymbol{R}_t.
\]
若用对数收益，需先转为简单收益再按权重加权，或在小收益下近似 \(R_{p,t} \approx \sum_i w_{i,t} r_{i,t}\)。

### 1.2 考虑分红与拆股

**定义 1.2.1（除权除息）**  
**除权除息**指分红或送股后价格下调；若用**未复权**价格，需在除权日对收益做调整（如加上每股分红再算收益），或直接使用**后复权价格**。

**定义 1.2.2（后复权价格）**  
**后复权**将历史价格按分红、拆股等比列调整，使收益率序列连续；回测中**优先用后复权**计算收益，避免断点。

**注意**：前复权与后复权差异在于基准日；回测常用后复权以保证「过去某日买入持有至今」的收益可连续计算。

### 1.3 多标的组合的收益序列

**定义 1.3.1（日度组合收益）**  
每日先确定**当日权重** \(\boldsymbol{w}_t\)（由上日末或当日开盘前确定，避免前视），再算
\[
R_{p,t} = \sum_i w_{i,t} R_{i,t}.
\]
**权重**可为等权、市值加权、信号加权等；**调仓日**更新权重，非调仓日可用「上一调仓日权重」或「考虑当日收益后的动态权重」（后者更精确但实现稍复杂）。

### 1.4 净值与累计收益

**定义 1.4.1（净值）**  
设初始净值 \(NAV_0 = 1\)，**净值递推**为
\[
NAV_t = NAV_{t-1} \times (1 + R_{p,t}).
\]

**定义 1.4.2（累计收益）**  
**累计收益** = \(NAV_T - 1\)，或 \((1+R_1)(1+R_2)\cdots(1+R_T) - 1\)。

**定义 1.4.3（年化收益）**  
若共 \(T\) 个交易日，假设 252 个交易日/年，**年化收益**为
\[
R_{\text{annual}} = (1 + R_{\text{cum}})^{252/T} - 1,
\]
其中 \(R_{\text{cum}}\) 为累计收益。更精确可用对数收益年化：\(r_{\text{annual}} = r_{\text{cum}} \times (252/T)\)，再转成简单收益 \(e^{r_{\text{annual}}}-1\)。

### 1.5 例题与代码

**例题 1.5.1**  
某组合 3 日简单收益为 0.01, -0.02, 0.015。初始净值 1，求每日净值与累计收益、年化收益（假设 3 日对应 3/252 年）。

**解**：  
- \(NAV_1 = 1\times(1+0.01)=1.01\)，\(NAV_2 = 1.01\times(1-0.02)=0.9898\)，\(NAV_3 = 0.9898\times(1+0.015)\approx 1.0046\)。  
- 累计收益 \(R_{\text{cum}} = 1.0046 - 1 = 0.0046\)。  
- 年化 \(R_{\text{annual}} = (1.0046)^{252/3} - 1\)（数值极大因外推；实际应用时 \(T\) 通常为数百日）。

**代码示例**（Python）：

```python
import numpy as np

def nav_series(returns: np.ndarray, nav0: float = 1.0) -> np.ndarray:
    """returns: 各期简单收益；返回净值序列"""
    return nav0 * np.cumprod(1 + np.asarray(returns))

def cum_return(returns: np.ndarray) -> float:
    return np.prod(1 + np.asarray(returns)) - 1

def annualize_return(returns: np.ndarray, trading_days_per_year: int = 252) -> float:
    r = np.asarray(returns)
    T = len(r)
    r_cum = np.prod(1 + r) - 1
    return (1 + r_cum) ** (trading_days_per_year / T) - 1

r = [0.01, -0.02, 0.015]
print("净值序列:", nav_series(r))           # [1.01, 0.9898, 1.004647]
print("累计收益:", cum_return(r))           # 约 0.00465
print("年化收益(3日):", annualize_return(r)) # 仅示例，T 小外推不稳定
```

---

### 本章小结（第 1 章）

1. **简单收益** \(R_t=(P_t-P_{t-1})/P_{t-1}\)；多期连乘得总收益。**对数收益** \(r_t=\ln(P_t/P_{t-1})\)，多期相加。  
2. **组合收益** \(R_{p,t}=\sum_i w_{i,t} R_{i,t}\)；**净值** \(NAV_t=NAV_{t-1}(1+R_{p,t})\)；用后复权价格。  
3. **年化** \((1+R_{\text{cum}})^{252/T}-1\)；实现时注意权重与收益时间对齐，避免前视。

## 第 2 章 滑点与手续费

**动机**：真实交易存在佣金、印花税、滑点与冲击；若回测中不扣除或低估，会高估策略收益与夏普，实盘无法复现。本节给出成本在收益中的体现方式与实现要点。

### 2.1 在收益中的体现

**定义 2.1.1（单笔交易成本）**  
**单笔交易**：买入时「多付」滑点+佣金，卖出时「少收」滑点+佣金+印花税（若有）；**净成本** = 买卖双方成本之和，常折成**单边费率**或**往返费率**（round-trip）。

**定义 2.1.2（组合当期成本）**  
**组合收益** = **毛收益 − 当期交易成本**；当期交易成本 = \(\sum\)（成交金额 × 费率 + 滑点 + 冲击估计），对当次调仓的所有成交汇总。

### 2.2 实现要点

**调仓日**：根据「目标权重」与「当前权重」的差，得到**交易量**（买入/卖出金额）；再按「交易金额 × 往返费率 + 滑点比例 + 冲击公式」计算成本。  

**成本从当日收益中扣除**：可假设成本在调仓日**当日**发生（保守）；或按实际成交日分摊。  

**无调仓日**：无交易则无成本；收益仅为持仓收益。

**公式（当期净收益）**  
设 \(R_{p,t}^{\text{gross}}\) 为按权重与标的收益算出的毛收益，\(C_t\) 为当日交易成本（调仓日为正，非调仓日为 0），则
\[
R_{p,t} = R_{p,t}^{\text{gross}} - C_t.
\]

### 2.3 敏感性分析

对**费率、滑点、冲击系数**做**多档假设**（如 0.5×、1×、2× 基准），看收益与夏普如何变化；若「1× 成本」下策略已失效，说明对成本敏感，实盘需格外注意执行与成本控制。

**例题 2.3.1**  
某日调仓买入 100 万元、卖出 80 万元；往返费率 0.1%（买卖各 0.05%），滑点 0.05%，无冲击。求当日交易成本。

**解**：买入成本 = 100×0.001 + 100×0.0005 = 0.15 万；卖出成本 = 80×0.001 + 80×0.0005 = 0.12 万（假设卖出含印花税在「费率」中）；总成本 = 0.27 万元。若组合规模 1000 万，则成本占组合约 0.027%。

---

### 本章小结（第 2 章）

1. **成本**：佣金+印花税+滑点+冲击；按调仓日交易金额计算，从**当日收益中扣除**；无调仓日无成本。  
2. **实现**：目标权重与当前权重差 → 交易金额 → 费率与滑点、冲击 → 扣减当日收益。  
3. **敏感性分析**：多档成本假设，评估策略对成本的稳健性。

## 第 3 章 前视偏差（Look-Ahead Bias）

**动机**：若在 \(t\) 时刻做决策时用到了 \(t\) 之后才可获得的信息，回测表现会虚高，实盘无法复现；前视偏差是回测失效的常见原因，必须从信息集与执行时点上严格杜绝。

### 3.1 含义

**定义 3.1.1（前视偏差）**  
**前视**指在 \(t\) 时刻做决策时，**用到了 \(t\) 时刻之后才可获得的信息**（如用到了 \(t\) 日收盘价来做 \(t\) 日开盘的决策、用到了未来数据做标准化等）。

**后果**：回测表现**虚高**，实盘无法复现。

### 3.2 常见来源

- **用当日收盘价调仓**：若信号基于「当日收盘」计算，则实盘无法在当日收盘前下单；应改为「T 日收盘信号 → T+1 日开盘或收盘执行」，或明确标注「可执行时点」。  
- **用未来数据标准化**：例如用**全样本**的均值与标准差做标准化，则每期都「看到」了未来；应使用**截至当期的历史**（滚动或扩展窗口）计算均值与标准差。  
- **用未来数据选股**：例如用了「当季财报」但财报公布日在期末之后；应用**公告日**或**可获知日**之后才能用该信息。  
- **幸存者偏差**：只用「当前仍存在的股票」回测，等价于用到了「未来是否退市」的信息（见第 4 章）。

### 3.3 避免方法

- **严格时间线**：每步写明「信息集」与「决策时点」；信号只用**该时点之前**的数据。  
- **标准化/参数**：均值、标准差、分位数、模型参数等，一律用**过去数据**（滚动或扩展）；不在任何地方用全样本或未来数据。  
- **执行假设**：明确「信号时点」与「成交时点」（如 T 日收盘信号、T+1 日开盘以开盘价成交），并在回测中**按此执行**。

**注意**：互斥易错——「T 日收盘价」在 T 日收盘那一刻才确定，因此「T 日收盘后」才能用该价格；若回测写的是「T 日以收盘价成交」，通常意味着 T 日收盘时点成交，与「T 日收盘信号」一致，但若实盘无法在收盘瞬间成交，应改为 T+1 开盘执行并用水洗开盘价或下一可用价。

---

### 本章小结（第 3 章）

1. **前视**：用到了 \(t\) 之后才可获得的信息；后果为回测虚高。  
2. **避免**：信号与标准化**只用过去数据**；执行在信号之后；不用全样本或未来数据做参数。  
3. **执行**：明确信号时点与成交时点（如 T 收盘信号 → T+1 开盘执行）。

## 第 4 章 幸存者偏差（Survivorship Bias）

**动机**：若回测只包含「当前仍上市」的股票，而忽略了已退市、被合并、被 ST 后剔除的股票，历史收益会被高估，回测与实盘差异大。

### 4.1 含义

**定义 4.1.1（幸存者偏差）**  
**幸存者偏差**指回测**只包含当前仍上市、仍存在于样本中的股票**，而**忽略了已退市、被合并、被 ST 后剔除**的股票。

**后果**：历史收益被**高估**（因表现极差的股票已被剔除）；回测与实盘差异大。

### 4.2 处理

- **使用「全历史成分」**：每期用**当时**的股票池（如当时已上市且未退市的全体，或当时的指数成分），而不是「用今天的成分回溯过去」。  
- **数据源**：使用包含**退市、摘牌**历史的数据；若只有当前成分，需标注「存在幸存者偏差」并谨慎解读结果。  
- **指数与基金**：同样存在「成分股调整」；若回测指数增强，应用**历史成分**而非当前成分。

---

### 本章小结（第 4 章）

1. **幸存者偏差**：仅用当前仍存在的股票回测会高估收益。  
2. **处理**：用**当时**的股票池或含退市数据，不用「当前成分回溯」。

## 第 5 章 样本内与样本外

**动机**：在样本内选因子、调参、选模型，再在样本外做一次评估，才能判断策略是否过拟合；若样本外数据参与过任何选择，则样本外也被「间接前视」，评估无效。

### 5.1 划分

**定义 5.1.1（样本内）**  
**样本内（In-Sample, IS）**：用于**估计参数、选因子、选模型、调参**的数据区间。

**定义 5.1.2（样本外）**  
**样本外（Out-of-Sample, OOS）**：**从未参与**任何选择与调参的数据，仅用于**最终评估**策略表现。

**原则**：必须先 IS 后 OOS；**禁止用 OOS 做任何模型或参数选择**，否则 OOS 被「间接前视」。

### 5.2 滚动与扩展

**定义 5.2.1（滚动窗口）**  
**滚动窗口**：只用**最近 \(T\) 期**数据训练，每期（或每 \(K\) 期）重新估计并预测；适应结构变化，但样本量固定。

**定义 5.2.2（扩展窗口）**  
**扩展窗口**：用**自起点至今**的全部历史训练，每期更新；样本量递增，更稳定但可能对近期变化反应慢。

回测中：**训练期 = IS**，**测试期 = OOS**；可做**多段 OOS**（如 2015–2016、2018–2019）看稳健性。

### 5.3 回测报告规范

- **明确**：IS 区间、OOS 区间、调仓频率、成本假设、是否含幸存者偏差、信号与执行时点。  
- **报告指标**：OOS 上的**年化收益、波动率、夏普、最大回撤、卡玛比、换手率、信息比**（若有基准）；必要时报告 IS 与 OOS 的对比，避免只在 IS 上「调参调出好结果」。

---

### 本章小结（第 5 章）

1. **IS**：训练与调参；**OOS**：从未参与选择，仅最终评估；严格时间顺序。  
2. **滚动/扩展**窗口；多段 OOS 验证稳健性。  
3. **报告**：IS/OOS、成本、前视与幸存者处理、信号与执行时点、核心指标。

## 第 6 章 回测检查清单（自用）

- [ ] 收益计算：复权价格、组合收益 = 权重 × 标的收益、净值递推正确。  
- [ ] 成本：佣金、印花税、滑点、冲击是否每笔/每期扣除；成本假设是否保守。  
- [ ] 前视：信号与标准化是否只用过去数据；执行是否在信号之后、且可执行。  
- [ ] 幸存者：股票池是否为历史成分或含退市。  
- [ ] 样本划分：OOS 是否严格未参与选因子/选模型/调参；是否多段 OOS 验证。  
- [ ] 其他：涨跌停与停牌是否过滤、T+1 是否遵守、流动性过滤（可选）。

---

## 第 7 章 与量化研究的衔接

- **策略开发**：在 IS 上开发与调参，在 OOS 上只做一次评估；若 OOS 表现与 IS 差距大，需警惕过拟合。  
- **实盘**：回测的「可执行规则」应尽量与实盘一致（委托方式、滑点与冲击假设）；实盘后可用**实盘收益**与**同期回测收益**对比，做**归因与改进**。  
- **论文与报告**：回测部分应写清上述要点，便于复现与审阅。

**建议**：能写出「权重×收益」的组合收益与净值递推；能列出至少三种前视偏差并说明如何避免；能说明幸存者偏差及应对；能设计「IS+OOS」的滚动或扩展回测流程，并写出一份简短的回测检查清单。
