# 数学公式显示说明

knowledge 下的文档使用 **LaTeX** 书写数学公式。若你的阅读环境中公式显示为**原始代码**（如 `\frac{1}{2}`、`\beta_0`）或**乱码**，是因为当前 Markdown 预览**没有启用数学公式渲染**。

---

## 一、公式写法说明

- **块级公式**：`$$ 公式 $$`，单独占一行。
- **行内公式**：`\( 公式 \)`，嵌在段落中。

例如：

```markdown
行内：\( E(X) = \mu \)，块级：
$$ \beta_1 = \frac{\widehat{\operatorname{Cov}}(X,Y)}{\widehat{\operatorname{Var}}(X)} $$
```

这些写法是标准 LaTeX/KaTeX/MathJax 语法，需要在**支持数学渲染**的 Markdown 预览中才能正确显示。

---

## 二、推荐解决方式

### 1. VS Code / Cursor 中正确显示公式

- **安装扩展**（任选其一）：
  - **Markdown+Math**（`goessner.mdmath`）：用 KaTeX 渲染，支持 `$$`、`$`、`\(\)`、`\[\]`。
  - **Markdown All in One** 等带数学支持的扩展。
- 安装后**重新打开 Markdown 预览**（或切换预览标签），公式应能正常显示。

### 2. 使用其他支持公式的编辑器

- **Typora**：打开 `.md` 文件即可渲染公式。
- **Obsidian**：默认支持 LaTeX 数学公式。
- **导出为 PDF**：用 Pandoc 等工具将 Markdown 转为 PDF，公式会正确渲染。

### 3. 在 GitHub 网页上查看

- **GitHub 的 .md 预览不渲染数学公式**，仓库里的 `.md` 在网页上会看到 LaTeX 源码而非公式。
- 建议**在本地**用上述编辑器或扩展打开 `knowledge` 下的文档查看；或克隆仓库后用 Typora/VS Code（带数学扩展）阅读。

---

## 三、小结

| 环境 | 是否默认显示公式 | 建议 |
|------|------------------|------|
| GitHub 网页 | 否 | 本地用支持公式的编辑器打开 |
| VS Code / Cursor 默认预览 | 视版本而定，常不完整 | 安装 Markdown+Math 等数学扩展 |
| Typora / Obsidian | 是 | 直接打开 .md 即可 |
| Pandoc 转 PDF/HTML | 是 | 导出后公式正常 |

**结论**：文档本身格式正确，无需修改。只需在**支持 LaTeX/KaTeX 的 Markdown 预览或编辑器中打开**即可避免“乱码”或源码式显示。
